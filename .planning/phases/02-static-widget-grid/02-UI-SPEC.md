## Spacing Scale

Standard 8-point scale. All grid and widget measurements are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon badge padding, size-toggle button internal padding (1px border + 1px inner = effectively 4px) |
| sm | 8px | Widget header internal gaps, check-list row gaps, match-card internal gaps |
| md | 16px | Widget body padding (14px horizontal, rounds to 16px in Tailwind — use `px-4`) |
| lg | 24px | Hub page main content horizontal padding (`px-6` / `px-7` per sketch) |
| xl | 32px | Spacing between grid sections (28px in sketch — use 28px as a named exception, see below) |
| 2xl | 48px | Major page section breaks (not used in Phase 2 hub layout) |
| 3xl | 64px | Not used in Phase 2 |

**Exceptions:**
- Widget card padding: `14px` vertical top / `16px` horizontal — use Tailwind `py-3.5 px-4` or `p-[14px_16px]`. Source: nav-hub-sketch.html `.widget { padding: 14px 16px }`.
- Grid gap: `14px` — use `gap-[14px]`. Source: nav-hub-sketch.html `.grid { gap: 14px }`.
- Section margin: `28px` bottom — use `mb-7`. Source: nav-hub-sketch.html `.section { margin-bottom: 28px }`.
- Widget icon container: `26px × 26px`, radius `7px` — use `w-[26px] h-[26px] rounded-[7px]`. Source: nav-hub-sketch.html `.w-icon`.
- Size-toggle button: `18px × 18px`, font `9px` — use `w-[18px] h-[18px] text-[9px]`. Source: nav-hub-sketch.html `.size-toggle button`.
- Touch targets for S/M/L toggle group: the toggle group as a whole must be at least `44px` wide to satisfy WCAG 2.1 AA touch target requirements. Three 18px buttons fill ~56px total; this requirement is met by the group.

**Product decision note:** The 14px/26px/18px non-multiples above are deliberate prototype-fidelity exceptions, not errors — do not "correct" them to 16/24/20px without re-validating the prototype.

## Typography

Four sizes, two weights. All roles map to the 32/22/13/12px scale.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| KPI large | 32px | 700 (bold) | 1.1 | Primary numeric KPI (e.g. "12 nya träffar") in S-mode or as featured stat in L-widget |
| Hub title / KPI medium | 22px | 700 (bold) | 1.2 / 1.1 | Hub page `<h2>` title in header (1.2); secondary KPI (e.g. "3 utkast", last score) (1.1) |
| Widget title | 13px | 700 (bold) | 1.3 | Widget card `<h3>` in `.w-title` |
| Body / label / sub-label | 12px | 400 (regular) | 1.5 | KPI labels, check-list items, match-card company names, section text, stack-bar legend, range bar labels, alert chips |

Source: nav-hub-sketch.html `.kpi-big { font-size: 32px }`, `.kpi-mid { font-size: 22px }`, `.w-title h3 { font-size: 13px }`, `.hero h2 { font-size: 22px }`, `.kpi-label { font-size: 12px }`, `.stack-legend { font-size: 11px }` (merged into 12px role).

**Declared sizes (4) and weights (2):**
- Sizes: 32px, 22px, 13px, 12px
- Weights: 400 (regular) and 700 (bold) — 600 is not used; widget titles previously at 13px/600 are 13px/700 (visual difference negligible at this size)

**Section heading:**
- 12px, weight 700, uppercase, letter-spacing 0.06em, color `--c-text`
- Differentiates from body/label (12px/400) via weight + uppercase + letter-spacing, not via a separate font size
- Source: nav-hub-sketch.html `.section-head h4` — size harmonized to 12px per typography constraint; uppercase + letter-spacing preserved for visual distinction

## Widget Grid Contract

Source: nav-hub-sketch.html `.grid` + `.widget.size-*` + ARCHITECTURE.md "Per-size rendering contract".

**Focal point declaration:** Primary visual anchor on Söka Jobb hub: CV widget at L-size, first position. The 32px KPI number is the primary eye-attractor on any hub section containing a numeric KPI.

### Grid Container

```
grid-template-columns: repeat(4, 1fr)      — desktop (≥ 900px)
grid-template-columns: repeat(2, 1fr)      — tablet/mobile (< 900px)
grid-auto-rows: 150px
gap: 14px
```

Use plain CSS grid (no react-grid-layout in Phase 2). The `HubGrid` component wraps this.

### Widget Size Specifications

| Size | Grid span | Row height | Class | Content rules |
|------|-----------|------------|-------|---------------|
| S | `span 1 col` × `span 1 row` | 150px | `col-span-1 row-span-1` | KPI number + label only. No footer. No visualization. |
| M | `span 2 col` × `span 1 row` | 150px | `col-span-2 row-span-1` | KPI + 2–3 lines context. 1 footer action max. |
| L | `span 2 col` × `span 2 rows` | 314px (2 × 150 + 14) | `col-span-2 row-span-2` | Full visualization (ring, list, sparkline). Up to 2 footer actions. |
| XL | `span 4 col` × `span 1 row` | 150px | `col-span-4 row-span-1` | Wide horizontal layout — onboarding banner, next-step CTA. |

**Mobile override (< 900px):**
- S: `col-span-1` stays (single column in 2-col grid)
- M, L: `col-span-2` (full width in 2-col grid)
- XL: `col-span-2` (full width in 2-col grid)
- L: `row-span-2` stays (two rows tall)

Source: nav-hub-sketch.html `@media (max-width: 900px)` block.

## Widget Component Contract

### Widget.Root

```
bg: var(--surface) = #FFFFFF
border: 1px solid var(--stone-150) = #EAE8E2
border-radius: 12px
padding: 14px 16px
display: flex; flex-direction: column
overflow: hidden
transition: border-color 150ms, box-shadow 150ms
```

Hover state: `border-color: var(--c-accent)` — no box-shadow by default; hover elevation is `0 2px 6px rgb(0 0 0 / 0.05)` (only on interactive hover, per docs/DESIGN.md).

### Widget.Header

Layout: `display: flex; align-items: center; justify-content: space-between; margin-bottom: 10px`.

Left side: icon container (26×26px, radius 7px, `bg: var(--c-bg)`, icon color `var(--c-text)`, aria-hidden) + `<h3>` at 13px/700 in `var(--stone-900)`.

Right side: size-toggle group (see below) — visible only when `editMode=true` OR on hover (opacity 0 → 1 at 150ms).

### Size Toggle Group

```
display: flex; gap: 1px
background: var(--stone-150)
border-radius: 6px
padding: 1px
opacity: 0  (default)
transition: opacity 150ms
```

On parent `.widget:hover` or `editMode=true`: `opacity: 1`.

Each toggle button:
```
width: 18px; height: 18px
font-size: 9px; font-weight: 700
border: none; background: transparent
color: var(--stone-700)
cursor: pointer
border-radius: 5px
```

Active toggle button:
```
background: #FFFFFF
color: var(--c-text)
```

**Keyboard accessibility (WCAG 2.1 AA — WIDG-02):**
- The three buttons (S, M, L) are native `<button>` elements — keyboard-operable by default.
- The toggle group receives `role="group"` with `aria-label="Välj widgetstorlek"`.
- Each button has `aria-pressed={size === activeSize}`.
- On size change, a `<div role="status" aria-live="polite" className="sr-only">` outside the grid announces: "Widgeten är nu [S/M/L]-storlek."
- Tab order: each widget's toggle group is reachable via Tab; arrow keys move between S/M/L within the group (`role="group"` + roving tabindex pattern).

Source: PITFALLS.md Pitfall 9, REQUIREMENTS.md WIDG-02, ARCHITECTURE.md Widget compound API.

### Widget.Body

```
flex: 1
display: flex; flex-direction: column
min-height: 0
```

Content varies by size (see Widget Size Specifications above and per-widget specs in Copywriting Contract).

### Widget.Footer

```
display: flex; gap: 6px
margin-top: 10px
```

Hidden in S-size. In M: max 1 button. In L: max 2 buttons.

Button variants in widget footers:
- Primary: `background: var(--c-solid); color: #fff; border-radius: 7px; padding: 6px 12px; font-size: 12px; font-weight: 700`
- Soft: `background: var(--c-bg); color: var(--c-text); border-radius: 7px; padding: 6px 12px; font-size: 12px; font-weight: 700`
- Link: `background: transparent; color: var(--c-text); font-size: 12px; font-weight: 700; padding: 6px 0`

Source: nav-hub-sketch.html `.btn`, `.btn-soft`, `.btn-link`.

## Error Boundary Fallback Card Contract

Each widget is wrapped in its own `ErrorBoundary`. When the boundary catches, the widget slot renders a fallback card in the same grid span/position as the original widget.

```
Same grid-column/grid-row span as the failed widget
border: 1px solid var(--stone-150)
border-radius: 12px
padding: 14px 16px
background: var(--surface)
display: flex; flex-direction: column; align-items: center; justify-content: center
gap: 8px
```

Content:
- Lucide `AlertCircle` icon, `16px`, color `var(--stone-500)`, `aria-hidden="true"`
- Heading: `"Kunde inte ladda"` — 13px/700/`var(--stone-700)`
- Body: `"Försök igen om en stund"` — 12px/400/`var(--stone-500)`
- Retry `<button>`: link-style, `font-size: 12px; color: var(--c-text)`, text `"Försök igen"`

The fallback card must not show the widget title, icon, or any data hint — it is a clean neutral slate. All surrounding widgets remain fully functional (WIDG-03).

Source: PITFALLS.md Pitfall 4, REQUIREMENTS.md WIDG-03.

## Animation & Motion Contract

Source: docs/DESIGN.md "Motion", tokens.css `--motion-*` tokens, PITFALLS.md Pitfall 13.

| Interaction | Duration | Easing | Reduced-motion behavior |
|-------------|----------|--------|------------------------|
| Widget card border-color on hover | `var(--motion-fast)` = 150ms | ease | No animation (instant) |
| Size-toggle opacity on hover | `var(--motion-fast)` = 150ms | ease | Instant (`opacity: 1` always when editMode=true) |
| Widget mount (framer-motion) | `var(--motion-standard)` = 300ms | ease-out | Skip animation entirely (`initial=false`) |
| Size change grid reflow | CSS grid reflow (browser-native) | — | No CSS transition on grid items (no `transition` on `.widget`) |

**prefers-reduced-motion rule:** `tokens.css` already sets `--motion-fast/standard/slow` to `0.01ms` under `@media (prefers-reduced-motion: reduce)`. Any animation using these tokens is automatically compliant. Framer Motion animations must check `useReducedMotion()` and pass `initial={false}` when true.

**Forbidden in this phase:** No auto-playing animations. No spin, float, or pulse on active content. Skeleton pulse (`animate-pulse`) is acceptable as a loading indicator.

## Bundle / Code-Split Contract

Every widget component MUST be `lazy()`-imported. No static import of any widget file in any hub page or HubGrid component.

```typescript
// CORRECT — in registry.ts
const CvWidget = lazy(() => import('./CvWidget'))

// FORBIDDEN — in JobsokHub.tsx or HubGrid.tsx
import { CvWidget } from './widgets/CvWidget'
```

Each widget slot in HubGrid is wrapped in `<Suspense fallback={<SkeletonCard />}>`.

Verification: `vite-bundle-visualizer` run after Phase 2 build must show zero widget component code in the main chunk. This is a Phase 2 acceptance criterion (ROADMAP.md Phase 2 success criterion 4).
